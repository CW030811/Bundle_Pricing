# ModelMart 产品调研与开发计划手册

> 版本：v1.0　|　更新：2026-06　|　定位：AI 能力超市 / Token API 中转站

---

## 一、市场与竞品调研

### 1.1 赛道概述
随着大模型爆发，**API 中转分发（中转站）** 成为一门成熟生意：站长聚合上游官方/批发 API，
统一为 OpenAI 兼容接口，加价或做负载均衡后二次分发给开发者。典型卖点为 **低价、稳定、高速**。

### 1.2 参考 Demo：memefast（魔因 API）
- **定位**：专业 AI 大模型 API 中转平台，聚合 GPT-5.3 / Claude Opus 4.6 / Gemini 3 Pro /
  DeepSeek / Grok 等 **550+ 模型**，按量计费，号称低至官方价 ~10%，99.9% 可用。
- **当前 MVP**：**仅销售 + 国内支付**（模型货架 + 充值），即本项目要复现的范围。

### 1.3 开源标杆
| 项目 | 技术栈 | 特点 |
| --- | --- | --- |
| **one-api**（songquanpeng） | Go + React | 中转站鼻祖，单二进制，渠道/令牌/计费 |
| **new-api**（QuantumNous） | Go + React | one-api 现代化分支，内置易支付/Stripe、缓存计费、多种授权登录、智能路由 |

**关键洞察**：one-api / new-api 只做「**纯模型中转**」。而本产品的目标是
**模型 + 智能体 + MCP + Skill 的超市**——后三类商品是开源方案不覆盖的**差异化空间**。

### 1.4 行业风险（来自调研）
中转站市场鱼龙混杂：存在「假模型、降智投毒、偷额度、隐藏倍率」等乱象。
对自建平台的启示：**计费透明、日志可查、渠道可控、对账闭环** 是赢得信任的核心。

## 二、产品定位与差异化

- **一句话**：像逛超市一样选购 AI 能力——模型、智能体、MCP、技能，明码标价、即买即用。
- **差异化**：
  1. **多品类货架**：不止模型，还有 Agent / MCP / Skill（开源中转站没有）。
  2. **计费透明**：统一额度单位 + 每次调用可查的用量日志。
  3. **五大板块闭环**：进货→调度→销售→统计→财务，一套后台管到底。

### 用户画像
| 角色 | 诉求 |
| --- | --- |
| 个人开发者 / 学生 | 低价、即开即用、一个 Key 调所有模型 |
| 中小团队 | 成本可控、用量可观测、令牌分权限 |
| 站长（你） | 多渠道调度、毛利可视、对账清晰、运营工具齐全 |

## 三、功能架构（五大板块）

| 板块 | 模块 | MVP | 说明 |
| --- | --- | :--: | --- |
| **进货** | 渠道管理 | ✅ | 上游供应商 Key、Base URL、模型映射、成本观测 |
| **调度** | 路由引擎 | ✅(基础) | 加权随机 + 优先级；失败重试/限流为后续 |
| **销售** | 商店/定价/注册/控制台/令牌 | ✅ | 用户全流程 |
| **销售** | 充值/支付/兑换码 | ✅(模拟) | 模拟支付 + 兑换码，预留易支付 |
| **统计** | 用户看板 + 管理看板 | ✅ | 余额/消费/调用趋势/Top 模型 |
| **财务** | 订单/对账/毛利 | ✅ | 订单流水、营收、毛利估算 |

## 四、MVP 范围（本期已交付）

✅ 注册/登录/RBAC　✅ 模型货架 + 定价页　✅ 令牌管理　✅ 充值（模拟）+ 兑换码
✅ OpenAI 兼容中转（鉴权/调度/计费/日志/流式）　✅ 用户看板　✅ 管理后台（用户/渠道/商品/订单/兑换码/公告 + 数据看板）

**MVP 明确不做**（留待后续）：真实支付商户对接、上游失败重试与熔断、Redis 缓存计费、邮件验证、多语言、OAuth 登录。

## 六、商业模式

- **加价分发**：上游成本价进货，零售价售出，赚 **差价毛利**（后台已含毛利估算）。
- **充值沉淀**：预充值 + 满赠形成资金沉淀与复购。
- **增值**：智能体/MCP/Skill 按次计费，毛利更高；企业版/私有化。

---

## 七、合规与风险

- **上游条款**：API 转售可能触及上游 ToS，需评估各供应商分发政策。
- **支付合规**：国内收款需商户资质；个人可先用易支付聚合，规模化需公司主体。
- **内容安全**：需接入敏感词/审计，留存日志以备合规。
- **信任**：坚持计费透明、不降智、不超扣，是长期经营的护城河。

## 八、技术决策记录（ADR 摘要）

| 决策 | 选择 | 理由 |
| --- | --- | --- |
| 自研 vs Fork | **自研** | 复现需自有可控；超市愿景超出 new-api 的纯模型边界 |
| 技术栈 | **Next.js 全栈** | 开发最快、部署最易、前后端一体，最适合 MVP 与迭代 |
| 数据库 | **SQLite + Prisma** | 零依赖起步，Schema 即换 Postgres |
| 支付 | **模拟 + 兑换码** | 无商户资质即可演示全流程，预留真实接口 |
| ORM 版本 | **Prisma 6**（非 7） | Prisma 7 改用驱动适配器，MVP 选稳定版降低风险 |
