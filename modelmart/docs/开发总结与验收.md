# ModelMart 开发总结与验收文档

> 项目：ModelMart 模型超市（Token API 中转站 MVP）
> 对标：[memefast.top](https://memefast.top)（魔因 API）的「销售 + 国内支付」MVP
> 完成时间：2026-06

---

## 一、用了什么工具栈

### 1. 运行/开发环境
- **Node.js v25 / npm v11**（已验证联网拉包正常）
- **Claude Code 内置能力**：
  - `WebSearch` / `WebFetch` —— 竞品与行业调研
  - **Claude Preview MCP**（`preview_start/screenshot/logs`）—— 托管 dev server、交付可访问预览链接、截图核对 UI
  - Next 16 离线文档（`node_modules/next/dist/docs/`）—— 适配破坏性变更

### 2. 应用技术栈
| 层 | 技术 |
| --- | --- |
| 全栈框架 | **Next.js 16**（App Router + Turbopack，前后端一体） |
| 语言 | **TypeScript**（`tsc --noEmit` 全量通过） |
| 样式/图标 | **Tailwind CSS v4** + **lucide-react**（自建深色科技风设计系统） |
| 数据库/ORM | **SQLite** + **Prisma 6**（9 张表，可平滑换 Postgres/MySQL） |
| 认证 | **jose**（JWT + HttpOnly Cookie）+ **bcryptjs** + **zod** |
| 路由守卫 | Next 16 **`proxy.ts`**（鉴权 + RBAC） |
| 部署 | **Docker / docker-compose**（SQLite 卷持久化）、Vercel |

### 3. 代码体量
- 后端接口文件 **19** 个（含通用后台 CRUD，实际覆盖 30+ 接口动作）
- 页面 **17** 个、复用组件 **7** 个、lib 模块 **8** 个、数据模型 **9** 张表
- TypeScript/TSX 约 **3800 行**

## 二、经过了几轮开发迭代

| 轮次 | 阶段 | 产出 |
| :--: | --- | --- |
| **第 0 轮** | 调研与方向决策 | 调研 memefast / one-api / new-api；把需求映射到「进货/调度/销售/统计/财务」五大板块；与用户敲定 4 项关键决策（自研 / Next.js 全栈 / 模拟支付+兑换码 / 本地预览+部署文档） |
| **第 1 轮** | 项目搭建 + 数据层 | `create-next-app` 脚手架；设计 9 张表的 Prisma Schema；编写种子数据（17 个商品、4 条渠道、演示账号、48 条用量日志） |
| **第 2 轮** | 后端 API | 鉴权 / 公开接口 / 令牌 / 充值 / 兑换码 / 日志 / 看板 / 后台 CRUD；**OpenAI 兼容中转端点**（鉴权→调度→计费→日志，支持流式） |
| **第 3 轮** | 前端 | 设计系统 + 商店首页 / 定价 / 文档 / 登录注册；用户控制台（概览/令牌/充值/账单/设置）；管理后台（看板/用户/渠道/商品/订单/兑换码/公告） |
| **第 4 轮** | 适配 + 联调验收 | 适配 Next 16 破坏性变更（`middleware→proxy`）、Prisma 7→6 回退；端到端冒烟测试；Preview 截图核对 UI |

## 三、最终实现了什么效果

- 一个 **可访问、可登录、能下单、能调用、能扣费、能对账** 的完整 AI 能力超市 MVP。
- 首页深色科技风、渐变 Hero、实时统计与能力货架；体验对标参考 demo 并在「多品类货架」上做了超越。
- **中转链路真实生效**：以演示令牌调用 `gpt-5.3`，正确返回 OpenAI 标准结构，并按 token 扣费（如 6+92 tokens → 扣 673 额度 ≈ ¥0.0067）、落库日志。
- **后台数据闭环**：管理员看板显示营收 ¥50、累计调用、Top 模型、7 日趋势、最近订单。

## 四、平台具备的功能清单

**用户侧**
- 注册（送 ¥2 体验额度）/ 登录 / 退出 / 改密 / 邀请码
- 模型货架（分类筛选）+ 定价页 + 接入文档
- 令牌管理：创建 / 限额 / 启停 / 删除 / 显隐 / 一键复制
- 充值：套餐 + 自定义 + 满赠 + 模拟扫码支付 + 兑换码
- 账单日志（分页）+ 控制台概览（余额/充值/消费/今日/7 日图表）

**管理侧（五大板块）**
- 数据看板（统计/财务）：营收、消费、毛利估算、调用量、Top 模型、近 7 天趋势、最近订单
- 用户管理（改余额/角色/封禁）
- 渠道管理（进货 + 调度：上游 Key/模型映射/权重/优先级/启停）
- 商品管理（模型/智能体/MCP/Skill 上下架与定价）
- 订单（财务对账）/ 兑换码批量生成 / 公告管理

**中转引擎**
- OpenAI 兼容 `/api/v1/chat/completions`
- 令牌鉴权 + 过期/限额校验 + 余额预检
- 加权随机 + 优先级的渠道调度
- 真实上游转发（OpenAI 兼容）或模拟回复
- 按 token 原子扣费 + 用量落库 + 流式 SSE 输出

## 六、如何访问 Demo

- **本地预览（即开即用）**：`http://localhost:3000`（开发服务器由 Claude Preview 托管运行）
- **演示账号**：管理员 `admin / admin1234`；用户 `demo / demo1234`
- **本地复跑**：
  ```bash
  cd modelmart && npm install && npm run db:push && npm run db:seed && npm run dev
  ```
- **上公网**：`docker compose up -d`（见 README 部署章节），或部署到 Vercel（生产换 Postgres）。
