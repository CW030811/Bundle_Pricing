## 4. 接入 MB 的单实例效果

### 4.1 严格 300s 单实例对比

单实例严格对比结果位于：

- `/Users/sensen/.openclaw/workspace/domains/revenue-management/experiments/cpbsd_fcp_pruned_mb_compare_n10k50_strict300/comparison_summary.json`

设置：

- 问题：`N = 10, K = 50`
- 时间上限：`300s`
- 方法：`FCP-pruned-MB` vs `BSP` vs `CPBSD-A`

结果：

| 方法 | In-sample | Out-of-sample | Runtime |
|---|---:|---:|---:|
| FCP-pruned-MB | 3.56187 | 2.39903 | 90.20s |
| BSP | 3.06674 | 2.45054 | 3.70s |
| CPBSD-A | 3.43482 | 2.85929 | 300.10s |

结论：

- `FCP-pruned-MB` 在 in-sample 上优于 `BSP` 与 `CPBSD-A`。
- 但在 out-of-sample 上明显弱于 `CPBSD-A`，仅略低于或接近 `BSP`。

### 5.2 平均结果

| 方法 | Avg In-sample | Avg Out-of-sample | Avg Runtime |
|---|---:|---:|---:|
| FCP-pruned-MB | 3.20733 | 2.35897 | 188.63s |

- `FCP-pruned-MB` 平均相对 `BSP` 的 in-sample 比例：`1.1773x`
- `FCP-pruned-MB` 平均相对 `CPBSD-A` 的 in-sample 比例：`1.0368x`
- `FCP-pruned-MB` 平均相对 `BSP` 的 out-of-sample 比例：约 `1.0271x`
- `FCP-pruned-MB` 平均相对 `CPBSD-A` 的 out-of-sample 比例：约 `0.8371x`

胜负统计：

- `FCP-pruned-MB` in-sample：`10/10` 胜 `BSP`
- `FCP-pruned-MB` in-sample：`9/10` 胜 `CPBSD-A`
- `FCP-pruned-MB` out-of-sample：`6/10` 胜 `BSP`
- `FCP-pruned-MB` out-of-sample：`0/10` 胜 `CPBSD-A`

候选空间压缩效果：

- `FCP-pruned-MB` 平均 bundle 空间大小：`33.4`
- 完整 bundle 空间大小：`1024`
- 平均只保留了约 `3.26%` 的空间

### 5.3 10-instance 结论

到目前为止，可以得出比较稳定的判断：

1. `MB x_kn GCN + FCP pruning` 对 MB 的训练内目标非常有效。
2. 它能在极小 bundle 空间下取得高质量的 in-sample 解。
3. 但从平均 out-of-sample 表现看，`CPBSD-A` 仍然明显更强。
4. 因此当前最准确的结论是：
   - `FCP-pruned-MB` 是一个有效的 MB 加速/近似候选方法；
   - 但它尚未在泛化收益上超过 `CPBSD-A`。
