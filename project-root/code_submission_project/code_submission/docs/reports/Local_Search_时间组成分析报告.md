# Local Search 时间组成详细分析报告

## 概述

本报告分析 Global Top-K 策略下，m10_n10、m20_n10、m30_n10 三个数据集的 Local Search 时间组成变化。

**注**: 当前实验中 K = ceil(sqrt(m*n))，即：
- m10_n10: K = 10
- m20_n10: K = 15
- m30_n10: K = 18

---

## 一、时间组成结构说明

```
Total Time
├── GCN Inference Time (threshold_time)        # GCN模型推理时间
├── Initial MILP Time (initial_milp_time)      # 初始MILP求解时间
└── Local Search Time (local_search_time)      # Local Search总时间
    ├── Initial LP Time                        # 初始LP求解时间
    ├── Iteration Time (total_iteration_time)  # 迭代总时间
    │   ├── Add Candidate Time                 # Add候选构建时间
    │   ├── Drop Candidate Time                # Drop候选构建时间
    │   ├── Neighbor Generation Time           # 邻域生成时间
    │   ├── LP Solve Time                      # LP求解总时间
    │   └── Neighbor Iteration Time            # 邻域遍历时间(不含LP)
    └── Final MILP Time                        # 最终MILP验证时间
```

---

## 二、总时间分解

### 表2.1: 各阶段绝对时间 (ms)

| 数据集 | Total Time | GCN Time | Initial MILP | Local Search | K值 |
|--------|------------|----------|--------------|--------------|-----|
| m10_n10 | **346.75** | 27.20 | 60.02 | 259.51 | 10 |
| m20_n10 | **2,095.10** | 27.58 | 425.66 | 1,641.87 | 15 |
| m30_n10 | **8,707.48** | 24.31 | 1,994.97 | 6,688.20 | 18 |

### 表2.2: 各阶段时间占比 (%)

| 数据集 | GCN占比 | Initial MILP占比 | Local Search占比 |
|--------|---------|------------------|------------------|
| m10_n10 | 7.8% | 17.3% | **74.8%** |
| m20_n10 | 1.3% | 20.3% | **78.4%** |
| m30_n10 | 0.3% | 22.9% | **76.8%** |

**结论**:
- Local Search 占总时间的 **75-78%**，是主要时间消耗
- Initial MILP 占 **17-23%**，随m增大占比略增
- GCN推理时间 **几乎恒定** (~27ms)，占比随m增大而下降

## 六、随m增长的时间变化

### 表6.1: 各时间组成的增长倍数 (以m10_n10为基准)

| 数据集 | m值 | Total | Initial MILP | LS Time | LP Time | LP调用次数 |
|--------|-----|-------|--------------|---------|---------|------------|
| m10_n10 | 10 | 1.0x | 1.0x | 1.0x | 1.0x | 1.0x |
| m20_n10 | 20 | **6.0x** | 7.1x | 6.3x | **5.9x** | 1.3x |
| m30_n10 | 30 | **25.1x** | 33.2x | 25.8x | **23.7x** | 1.8x |

### 表6.2: 增长率分析

| 指标 | m: 10→20 | m: 20→30 | 趋势 |
|------|----------|----------|------|
| Total Time | 6.0x | 4.2x | 超线性但减缓 |
| Initial MILP | 7.1x | 4.7x | 超线性 |
| LP总时间 | 5.9x | 4.0x | 超线性 |
| LP调用次数 | 1.3x | 1.4x | **接近线性** |
| 单次LP时间 | 4.5x | 2.9x | 超线性 |

**关键洞察**:
1. **LP调用次数增长较慢** (1.3x-1.4x)，这是Global Top-K策略的优势
2. **单次LP求解时间增长快** (4.5x-13x)，是时间增长的主因
3. **总时间增长主要由LP求解复杂度驱动**，而非LP调用次数

## 八、核心结论

### 8.1 时间组成特点

1. **LP求解是绝对瓶颈**: 占Iteration时间的96-98%
2. **候选构建极快**: Add/Drop Candidate + Neighbor Gen 总共 < 2%
3. **GCN推理时间恒定**: ~27ms，不随m变化

### 8.2 随m增长的变化

| 变化趋势 | 说明 |
|----------|------|
| LP调用次数 | **缓慢增长** (O(sqrt(m))效果显现) |
| 单次LP时间 | **快速增长** (LP问题规模增大) |
| 候选构建时间 | **线性增长** 但绝对值很小 |
| 总时间 | **超线性增长** (主要由LP复杂度驱动) |

### 8.3 优化建议

| 方向 | 预期效果 | 难度 |
|------|---------|------|
| 减少LP调用次数 (更小的K) | 显著 | 低 (可能牺牲收益) |
| LP求解器优化 (并行、预处理) | 中等 | 中 |
| LP增量求解 (Warm Start) | 中等 | 高 |
| 降低MILP精度要求 | 中等 | 低 |
