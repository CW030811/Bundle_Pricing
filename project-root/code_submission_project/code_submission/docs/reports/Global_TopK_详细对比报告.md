# Global Top-K 策略详细对比报告

## 概述

本报告对比了三种Local Search策略在8个数据集上的实验结果：
1. **原策略 (Segment-based)**: 每轮生成 2*m 个邻域
2. **Global Top-K (K=2*sqrt(m))**: 每轮生成最多 2*K 个邻域，K = ceil(2*sqrt(m))
3. **Global Top-K (K=sqrt(m))**: 每轮生成最多 2*K 个邻域，K = ceil(sqrt(m))

---

## 十、关键发现

### 10.1 性能优势

1. **时间效率显著提升**: 平均加速3.2倍，最高可达6倍
2. **LP调用大幅减少**: 平均减少73.5%，是性能提升的核心原因
3. **可扩展性好**: 随着问题规模增大，优势更明显

### 10.2 收益表现

1. **收益差异可控**: 平均仅下降1.56%
2. **在常规数据集上差异极小**: m_n数据集上差异<0.25%
3. **BSP数据集差异稍大**: 约1.5%-3.1%

### 10.3 权衡分析

| 场景 | 推荐策略 | 理由 |
|------|---------|------|
| 对时间敏感 | K=sqrt(m) | 时间减少65%+，收益仅损失1.5% |
| 对收益敏感 | K=2*sqrt(m) | 收益基本持平，时间减少30%+ |
| 大规模问题 | K=sqrt(m) | 加速效果更显著，可达5-6倍 |

---

## 十一、K值选择建议

| K值公式 | 优点 | 缺点 | 适用场景 |
|---------|------|------|---------|
| K=sqrt(m) | 最大加速，LP调用最少 | 收益略有下降 | 对时间敏感的应用 |
| K=2*sqrt(m) | 收益接近原策略 | 加速较小 | 对收益敏感的应用 |
| K=sqrt(m*n) | 考虑产品数量 | 待验证 | 产品数量变化大的场景 |

---

## 十二、结论

**Global Top-K策略（K=sqrt(m)）**：

1. **时间效率**: 平均加速3.2倍，在大规模问题上可达6倍
2. **收益损失**: 平均仅损失1.56%
3. **核心机制**: 通过全局排序选择最有希望的邻域，大幅减少LP调用
4. **可扩展性**: O(sqrt(m))的邻域复杂度，适合大规模问题

**推荐**: 在大多数应用场景下，建议使用K=sqrt(m)策略以获得最佳的时间-收益权衡。
