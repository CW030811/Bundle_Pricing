## 五、实验数据对象调整指南

### 数据集路径配置
所有测试脚本的数据集路径通常在以下位置配置:
- `test_data_path = './dataset/test_data/'` 或
- `dataset_base_dir = os.path.join(script_dir, "Dataset")`

### 数据集格式
数据集文件为 `.msgpack` 格式，包含以下字段:
- `product_num`: 产品数量 (n)
- `segment_num`: 客户段数量 (m)
- `unit_cs`: 单位成本 (1, n) 或 (n,)
- `ship_cs`: 运输成本 (m, 1) 或 (m,)
- `unit_us`: 单位效用 (m, n)
- `Ns`: 客户段大小 (m, 1) 或 (m,)
- `opt_bundles`: 最优bundle分配
- `opt_prices`: 最优价格
- `opt_rev`: 最优收入
- `running_time`: 运行时间
- `gap`: 最优性间隙
- `cs`: 成本矩阵 (m, 2^n) [可选，旧格式]
- `Rs`: 估值矩阵 (m, 2^n) [可选，旧格式]

### 关键参数调整

#### 1. 随机种子设置
在数据生成脚本中:
```python
np.random.seed(53)  # 设置随机种子
# 对于Gurobi，在模型创建后:
model.setParam('Seed', 42)
```

#### 2. 模型路径
```python
model_path = "./best_model_edge.pt"  # 或使用绝对路径
model_path = os.path.join(script_dir, "best_model_edge.pt")
```

#### 3. 数据集选择
在 `test_FCP_LS.py` 和 `LS_Path_Test.py` 中:
```python
datasets = {
    'dataset_name': 'path/to/dataset',
    # 添加更多数据集
}
```

#### 4. 样本数量限制
```python
max_samples_per_dataset = 1000  # 每个数据集最多处理的样本数
actual_test_count = min(sample_num, max_samples_per_dataset)
```

#### 5. Local Search参数
```python
max_iterations = 50      # 最大迭代次数
tolerance = 1e-6         # 收入改进容忍度
K = int(ceil(sqrt(m)))   # Top-K参数 (仅LS_Path_Test.py)
```

#### 6. MILP求解器参数
```python
model.Params.OutputFlag = 0      # 关闭输出
model.Params.MIPGap = 1e-2       # MIP间隙
model.Params.TimeLimit = 600     # 时间限制(秒)
```

#### 7. LP求解器参数
```python
model.setParam("OutputFlag", 0)
model.setParam("Method", -1)     # 自动选择方法
model.setParam("Presolve", 2)    # 预处理级别
model.setParam("Threads", 1)     # 线程数
model.Params.TimeLimit = 300
```

## 七、注意事项

1. **当前默认环境**: 优先使用 `code_submission/.venv`
2. **历史命令说明**: 若看到 `conda activate pyg311` 或根目录 `python test_*.py`，应将其视为旧工作流记录
3. **编码设置**: Windows系统可按需使用 `chcp 65001` 设置UTF-8编码
4. **路径格式**: 使用 `os.path.join()` 确保跨平台兼容性
5. **随机种子**: 实验可复现性要求，必须设置并记录随机种子
6. **Gurobi许可证**: 确保Gurobi许可证已正确配置

## 八、输出文件说明

### CSV结果文件列说明
- `n_products`: 产品数量
- `revenue_ratio`: 收入比率（相对于最优解）
- `runtime_ratio`: 运行时间比率（相对于基准方法）
- `total_time`: 总运行时间
- `base_running_time`: 基准运行时间
- `threshold_time`: GCN推理时间
- `initial_milp_time`: 初始MILP求解时间
- `local_search_time`: Local Search时间
- `initial_revenue`: 初始收入比率
- `improvement`: 改进幅度
- `iterations`: 迭代次数
- `improvements`: 改进次数
- `lp_solver_calls`: LP求解器调用次数
- `milp_solver_calls`: MILP求解器调用次数
- `K`: Top-K参数值 (仅Global Top-K策略)
- `max_neighbors_per_iter`: 每次迭代最大邻域数 (仅Global Top-K策略)
