## 六、指标定义

### 6.1 Relative Difference (相对差异)

以 **K=sqrt(m)** 为基准策略，计算其他策略相对于基准的相对差异：

**Revenue Difference (%)**:
```
Revenue Diff = (Strategy_Revenue - Baseline_Revenue) / Baseline_Revenue × 100%
```
- **正值**：表示该策略的 Revenue Ratio 高于基准（更好）
- **负值**：表示该策略的 Revenue Ratio 低于基准（更差）

**Time Difference (%)**:
```
Time Diff = (Baseline_Time - Strategy_Time) / Baseline_Time × 100%
```
- **正值**：表示该策略的 Time Ratio 低于基准，即运行更快（更好）
- **负值**：表示该策略的 Time Ratio 高于基准，即运行更慢（更差）

相对差异图展示了各策略相对于 K=sqrt(m) 的表现，可以直观看出：
- K=sqrt(m) 相对于 2m 和 K=sqrt(mn) 在 Time 上的优势（Time Diff 为正值）
- K=sqrt(m) 相对于常数值（K=5, K=10）在 Revenue 上的表现（Revenue Diff）

### 6.2 Efficiency Ratio (效率比值)

效率比值定义为 Revenue Ratio 与 Time Ratio 的比值：

```
Efficiency = Revenue_Ratio / Time_Ratio
```

**含义**：
- **值越大越好**：表示单位时间成本下获得更高的收益
- 该指标综合了 Revenue 和 Time 两个维度，用于评估策略的整体效率
- 当 Revenue Ratio 相近时，Time Ratio 越小，效率比值越大

效率比值图展示了各策略在不同数据集上的效率表现，可以直观看出 K=sqrt(m) 在平衡 Revenue 和 Time 方面的优势。
