# 项目介绍：Mixed Bundling Pricing（code_submission）

## 1. 项目定位
这是一个**混合捆绑定价（Mixed Bundling Pricing）**研究与实验项目，核心目标是在给定客户分群与商品信息的情况下，求解能最大化收益的捆绑方案与价格。

项目采用了“**机器学习预测 + 数学优化求解 + 局部搜索加速**”的混合技术路线：
- 先用 GCN（图卷积网络）预测商品-客户分群关联概率；
- 再用 MILP / LP 做可行优化与收益评估；
- 最后通过局部搜索（含 Global Top-K 策略）提升求解效率与效果。

---

## 2. 主要技术路线

### 2.1 端到端流程
1. 读取数据集（`.msgpack`）并构建二部图（商品节点 + 客群节点）；
2. GCN 输出概率矩阵 `P[m,n]`；
3. 阈值法生成初始捆绑预测；
4. MILP 求初始可行最优分配；
5. LP 在邻域内快速评估，进行贪心式局部搜索；
6. 最终回到 MILP 进行验证与收敛结果确认。

### 2.2 关键策略
- **FCP**：固定阈值的初始选择策略（基线）
- **PCP**：渐进式候选策略
- **BSP**：Bundle Size Pricing 相关策略
- **Global Top-K Local Search（亮点）**：
  - `LS_Path_Test.py` 中将每轮邻域由传统 `2*m` 缩减为 `2*K`，其中 `K = ceil(sqrt(m))`；
  - 在保持效果的前提下，减少 LP 调用规模，提升搜索效率。

## 3. 代码与目录结构（核心）

- `src/test/test_FCP_LS.py`：核心框架与通用函数（模型、MILP/LP、局部搜索主流程）
- `src/test/LS_Path_Test.py`：Global Top-K 局部搜索实现（重点创新）
- `src/test/test_FCP.py` / `src/test/test_PCP.py` / `src/test/test_BSP.py`：不同策略实验入口
- `src/train/Training_edge-final.py`：GCN 模型训练脚本
- `src/data/run_cpbsd_baselines_v2.py`：当前 CPBSD smoke / sanity-check 主入口
- `src/data/run_cpbsd_main_n5_v2.py`：计划中的 N=5 v2 主实验脚本
- `best_model_edge.pt`、`models_multi_layer_edge_update/`：模型权重
- `Dataset/`、`dataset2_4_2026/`：实验数据集
- `experiments/`：当前 CPBSD / GCN / labeling / coverage 结果根目录
- `gurobi_stats_logs/`：求解器日志
- `analyze_*.py`、`compare_*.py`：结果统计与对比分析脚本
- 多个 `*.csv / *.png / *.md`：实验产出与分析报告

### 价值
- 将 GCN 与 OR（MILP/LP）有效结合，适合“预测 + 决策”一体化场景；
