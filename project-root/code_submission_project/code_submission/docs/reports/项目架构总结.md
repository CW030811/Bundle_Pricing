# Bundle定价优化项目架构总结

## 一、项目背景

这是一个**运筹优化**项目，研究的是**混合捆绑定价问题（Mixed Bundling Pricing Problem）**。核心目标是为不同的客户细分（segments）确定最优的产品组合（bundles）和定价策略，以最大化收益。

## 二、核心问题定义

### 输入参数
- `m`: 客户细分数量（segments）
- `n`: 产品数量（products）
- `unit_cs`: 单位成本矩阵
- `ship_cs`: 运输成本
- `unit_us`: 客户效用矩阵
- `Ns`: 各细分市场规模

### 目标
找到每个客户细分的最优bundle分配和定价，使得总收益最大化。

## 三、核心策略对比

| 策略 | 候选集生成 | 优化方法 | 特点 |
|------|-----------|---------|------|
| **FCP** (test_FCP.py) | 阈值方法 (≥0.0) | 单次MILP | 简单直接 |
| **PCP** (test_PCP.py) | Top-M选择 + 渐进bundle | 单次MILP | 利用结构约束 |
| **FCP+LS** (test_FCP_LS.py) | 阈值初始 + 逐segment邻域 | 迭代LP+MILP | 2*m邻域/轮 |
| **Global Top-K** (LS_Path_Test.py) | 阈值初始 + 全局Top-K | 迭代LP+MILP | 2*K邻域/轮, K=√m |

## 五、Global Top-K策略详解（核心创新）

### 核心思想
不再按segment逐个生成邻域，而是全局选择最优候选：

```python
K = ceil(sqrt(m))  # 关键参数

# Add候选：选择概率最高的K个未选产品
add_list = sorted([(k,j,P[k,j]) for未选产品], by=概率, 降序)[:K]

# Drop候选：选择概率最低的K个已选产品（且P≥0.5）
drop_list = sorted([(k,j,P[k,j]) for已选产品 if P≥0.5], by=概率, 升序)[:K]

# 每轮最多生成 2*K 个邻域（比原策略的 2*m 少很多）
```

### 邻域规模对比
- **原策略**: 每轮 2*m 个邻域（线性增长）
- **Global Top-K**: 每轮 2*K 个邻域，K=√m（次线性增长）
