# 求解空间对比实验报告

> 注：本文件记录的是一轮历史求解空间分析实验。文中的 Windows 环境、`pyg311`、`analyze_solution_space.py` 根目录命令主要用于回溯旧工作流，不再代表当前默认入口；当前默认环境是 `code_submission/.venv`，脚本路径应优先按 `src/...` 结构理解。

## 1. 实验目的

本实验旨在对比分析Mixed Bundling Pricing Problem（MBPP）中三种不同策略的求解空间特征，包括：
- **MILP策略**：使用所有可能的bundle（2^n个）
- **FCP策略**：Fixed Choice Prediction，使用GCN预测的bundle
- **PCP策略**：Progressive Choice Prediction，使用progressive bundle chains

通过统计和对比以下关键指标，评估各策略在问题规模缩减和求解效率方面的表现：
1. Bundle Space Size（Bundle数量）
2. Total Variables（总变量数）
3. Total Constraints（总约束数）
4. NumNZs（约束矩阵非零元素数，反映稀疏性）
5. Subadditivity Constraints（子可加性约束数量）
6. Model Build Time（模型构建时间）

## 2. 实验设计

### 2.1 实验样本
- **数据集**：m10n10、m20n10、m30n10
- **样本数量**：每个数据集取前50个样本
- **样本来源**：`Dataset/m10_n10_sample_100/`、`Dataset/m20_n10_sample_100/`、`Dataset/m30_n10_sample_100/`
- **样本特征**：
  - 产品数（n）：10
  - 客户段数（m）：10 / 20 / 30
  - 理论Bundle数量：2^10 = 1,024

### 2.2 实验环境
- **操作系统**：Windows 10
- **Python环境**：Conda环境 `pyg311` (Python 3.11)
- **关键依赖**：
  - PyTorch（GCN模型推理）
  - Gurobi（MILP/LP求解器）
  - NumPy, Pandas, Matplotlib（数据处理与可视化）
  - msgpack（数据序列化）

### 2.3 实验脚本
- **脚本文件**：`analyze_solution_space.py`
- **功能**：对三种策略分别构建MILP模型（不求解），通过Gurobi模型属性统计各项指标
- **运行命令**：`python analyze_solution_space.py --batch m10n10,m20n10,m30n10 50`

## 3. 实验方法

### 3.1 指标获取方式

本实验采用**Gurobi模型属性**直接获取求解空间复杂度指标，无需手动计数：

| 指标 | Gurobi属性 | 说明 |
|------|------------|------|
| Total Variables | `model.NumVars` | 总变量数 |
| Total Constraints | `model.NumConstrs` | 总约束数 |
| NumNZs | `model.NumNZs` | 约束矩阵非零元素数 |
| NumBinVars | `model.NumBinVars` | 二进制变量数 |
| Subadditivity Constraints | 约束名称前缀 `subadd_` 统计 | 子可加性约束数量 |

子可加性约束通过在添加约束时使用统一命名前缀（`subadd_`），在模型构建完成后通过 `model.getConstrs()` 遍历统计实现。

### 3.2 MILP策略
参考`generate_data_MB.py`中的`solve_bundle_MILP`函数：
- **Bundle空间**：使用所有2^n = 1,024个可能的bundle
- **变量**：p[B], theta[m,B], s[m], S[m,B], Z[m,B], P[m,B]
- **约束**：剩余效用、子可加性、IC、定义约束

### 3.3 FCP策略
参考`test_FCP.py`中的`revenue_ratio`函数：
- **Bundle空间**：GCN预测每个客户段的产品选择，提取唯一bundle
- **GCN**：`best_model_edge.pt`，阈值 logits >= 0.0
- **模型**：仅针对预测bundle创建变量和约束

### 3.4 PCP策略
参考`test_PCP.py`中的`revenue_ratio`函数：
- **Bundle空间**：progressive bundle chains
- **生成**：GCN → Top-M选择(threshold=0.5) → progressive bundles
- **约束**：Progressive chain约束 + Cross-segment约束（含`_is_progressive_redundant`过滤）

## 5. 结果分析

### 5.1 Bundle Space Size

- **MILP**：固定为1,024（完整bundle空间）
- **FCP**：随m增加，约10～28个bundle，较MILP缩减约97%～99%
- **PCP**：约64～161个bundle，较MILP缩减约84%～94%

**结论**：FCP和PCP均显著缩小bundle空间，FCP缩减最彻底。

### 5.2 Total Variables

- **MILP**：41,994（m10）～123,934（m30），与m×B近似成正比
- **FCP**：409～3,471，约为MILP的1%～3%
- **PCP**：2,628～19,557，约为MILP的6%～16%

**结论**：变量数量与bundle数量高度相关，FCP与PCP均明显减少变量规模。

### 5.3 Total Constraints

- **MILP**：84,307～187,567，随m单调增加
- **FCP**：775～10,194，约为MILP的1%～5%
- **PCP**：13,850～254,673，在m30时超过MILP（约136%）

**结论**：PCP在m较大时约束数超过MILP，主要来自progressive chain与cross-segment约束。

### 5.4 NumNZs（非零元素）

- **MILP**：457,341～2,402,961
- **FCP**：4,024～79,823，约为MILP的1%～3%
- **PCP**：53,883～1,051,388，约为MILP的12%～44%

**结论**：NumNZs与约束和变量规模相关，FCP矩阵最稀疏。

### 5.5 Subadditivity Constraints

- **MILP**：固定32,977（n=10时子可加性约束数）
- **FCP**：168～4,968，约为MILP的0.5%～15%
- **PCP**：10,527～229,476，在m30时约为MILP的696%

**结论**：PCP因progressive chain和cross-segment约束，子可加性约束数量随m快速增加。

### 5.6 Model Build Time

- **MILP**：3.07s（m10）～15.91s（m30）
- **FCP**：0.03s～0.46s，较MILP缩短约93%～97%
- **PCP**：0.30s～5.55s，较MILP缩短约65%～90%

**结论**：FCP构建时间最短，PCP次之，两者均快于MILP。

## 6. 关键发现

### 6.1 策略对比

| 维度 | MILP | FCP | PCP |
|------|------|-----|-----|
| Bundle空间 | 完整 | 最小 | 中等 |
| 变量数 | 最大 | 最小 | 中等 |
| 约束数 | 中等 | 最小 | m大时最大 |
| 子可加性约束 | 固定 | 最少 | m大时最多 |
| 构建时间 | 最长 | 最短 | 中等 |

### 6.2 实际应用建议

1. **追求速度**：选择FCP，构建时间最短
2. **保留结构化bundle**：选择PCP，但需接受更高约束复杂度
3. **完整求解**：选择MILP，覆盖全部bundle组合

## 7. 实验局限性

1. **仅统计构建时间**：未包含Gurobi求解时间
2. **GCN依赖**：FCP和PCP表现依赖GCN预测质量
3. **n=10固定**：结论在更大n下的扩展性待验证

## 8. 结论

本实验基于Gurobi模型属性对MILP、FCP、PCP三种策略的求解空间进行了对比：

1. **FCP策略**在问题规模缩减和构建时间上表现最佳，变量、约束和构建时间均显著低于MILP。
2. **PCP策略**在保留结构化bundle的同时，在m较大时约束数量超过MILP，构建时间仍显著短于MILP。
3. **MILP策略**使用完整bundle空间，规模最大，构建时间最长。
