# 策略核心逻辑伪代码

本文档记录三个策略代码的核心逻辑：FCP策略、PCP策略、Global Top-K Local Search策略。

---

## 1. FCP策略 (Fixed Candidate Products)

### 伪代码

1. 数据预处理
   - 加载GCN模型
   - 处理测试数据，构建图结构

2. GCN推理
   - 对图数据进行前向传播
   - 得到logits矩阵 logits_nm (shape: n × m)
   - 转换为预测矩阵: pred_assort = (logits_nm.T >= 0.0).astype(int)
     // pred_assort[i, j] = 1 表示segment i选择product j

3. 生成候选bundle集合
   - 对每个segment k，将pred_assort[k, :]转换为bundle_id
   - 收集所有唯一的bundle_id作为候选bundle集合

4. MILP优化
   - 仅对候选bundle集合计算Rs和cs矩阵
   - 构建MILP模型，变量仅包含候选bundle
   - 添加标准约束和子可加性约束
   - 求解MILP，得到最优收益

5. 计算收益比率
   - revenue_ratio = MILP最优收益 / 真实最优收益
   - 返回结果
```

### 关键特点
- **固定候选集**：基于阈值0.0直接确定产品选择
- **简单直接**：无迭代优化，一次MILP求解
- **计算高效**：仅对预测bundle计算成本矩阵

## 2. PCP策略 (Progressive Candidate Products)

### 伪代码

```
算法: PCP策略
输入: 测试数据样本
输出: 收益比率 (revenue_ratio)

1. 数据预处理
   - 加载GCN模型
   - 处理测试数据，构建图结构

2. GCN推理
   - 对图数据进行前向传播
   - 得到logits矩阵，转换为概率: prob = sigmoid(logits_nm)
     // prob[i, j] 表示product i在segment j中的选择概率

3. Top-M产品选择 (M = n, threshold = 0.5)
   FOR each segment k:
       - 按概率降序排序所有产品
       - 选择前n个产品
       - 过滤: 仅保留概率 >= 0.5的产品
       - 得到selected_products[k] (按概率降序排列)

4. 生成渐进式bundle集合
   feasible_bundles = {0}  // 包含空bundle
   
   FOR each segment k:
       selected_list = selected_products[k]
       FOR i = 1 to len(selected_list):
           // 生成渐进bundle: {p1}, {p1,p2}, {p1,p2,p3}, ...
           bundle = {selected_list[0], ..., selected_list[i-1]}
           bundle_id = binary_to_int(bundle)
           feasible_bundles.add(bundle_id)

5. 优化的MILP求解
   - 仅对feasible_bundles计算Rs和cs矩阵
   - 构建MILP模型
   
   // 添加渐进链约束（Progressive Chain Constraints）
   FOR each segment k:
       progressive_bundles = [bundle_1, bundle_2, ..., bundle_M]
       // bundle_i = {p1, ..., pi}
       FOR i = 1 to len(progressive_bundles) - 1:
           ADD CONSTRAINT: p[bundle_i] <= p[bundle_{i+1}]
   
   // 添加跨segment子可加性约束（Cross-Segment Constraints）
   FOR each bundle k in feasible_bundles:
       FOR each bundle i, j in feasible_bundles (i < j):
           IF k ⊆ (i ∪ j) AND k ≠ i AND k ≠ j:
               IF NOT is_progressive_redundant(k, i, j):
                   ADD CONSTRAINT: p[k] <= p[i] + p[j]
   
   // 添加标准约束
   - 添加价格、收益、选择等标准约束
   - 求解MILP，得到最优收益

6. 计算收益比率
   - revenue_ratio = MILP最优收益 / 真实最优收益
   - 返回结果
```

### 关键特点
- **渐进式bundle**：每个segment生成链式bundle {p1} ⊆ {p1,p2} ⊆ ...
- **链式约束**：利用bundle包含关系减少约束数量
- **Top-M过滤**：仅考虑高概率产品（prob >= 0.5）
- **约束优化**：避免渐进链内部的冗余约束

## 3. Global Top-K Local Search策略

### 核心思想
使用阈值方法生成初始解，通过全局Top-K邻域搜索进行局部优化。

### 伪代码

```
算法: Global Top-K Local Search策略
输入: 测试数据样本, max_iterations=50, tolerance=1e-6
输出: 最终收益比率 (final_revenue_ratio), 搜索信息

1. 初始bundle预测
   - 使用阈值方法生成initial_pred_assort
   - 使用GCN得到概率矩阵prob (用于邻域生成)

2. 初始MILP求解
   - 对initial_pred_assort进行MILP求解
   - 得到initial_assignment和initial_milp_ratio

3. 初始LP求解
   - 对initial_assignment进行LP可行性检查
   - 得到current_revenue和current_assignment

4. Local Search主循环
   K = ceil(sqrt(m))  // 全局Top-K参数
   iteration = 0
   improved = True
   
   WHILE improved AND iteration < max_iterations:
       improved = False
       iteration = iteration + 1
       
       // 生成全局Top-K邻居
       neighbors = Generate_Global_TopK_Neighbors(
           current_assignment, prob, n, m, K
       )
       // 最多生成2*K个邻居
       
       // 评估邻居（按优先级顺序）
       FOR each neighbor_assignment in neighbors:
           // LP可行性检查和收益评估
           is_feasible, neighbor_revenue = LP_Check(neighbor_assignment)
           
           IF is_feasible AND neighbor_revenue > current_revenue + tolerance:
               // 贪婪策略：立即接受改进
               current_assignment = neighbor_assignment
               current_revenue = neighbor_revenue
               improved = True
               BREAK  // 找到改进后立即跳出
       
       IF NOT improved:
           // 未找到改进，搜索收敛
           BREAK

5. 生成全局Top-K邻居的详细逻辑
   函数: Generate_Global_TopK_Neighbors(current_assignment, prob, n, m)
   
   // 转换为矩阵形式
   current_pred_assort = assignment_to_matrix(current_assignment, n, m)
   K = ceil(sqrt(m))
   neighbors = []
   
   // Step 1: 生成Add候选（全局排序）
   add_candidates = []
   FOR k = 0 to m-1:
       FOR j = 0 to n-1:
           IF current_pred_assort[k, j] == 0:  // 当前未选择
               score = prob[k, j]  // 概率越高，越优先添加
               add_candidates.append((k, j, score))
   
   // 按概率降序排序
   sort(add_candidates, by=score, descending=True)
   add_list = add_candidates[:K]  // 取Top-K
   
   // Step 2: 生成Drop候选（全局排序）
   drop_candidates = []
   FOR k = 0 to m-1:
       FOR j = 0 to n-1:
           IF current_pred_assort[k, j] == 1:  // 当前已选择
               score = prob[k, j]  // 概率越低，越优先删除
               drop_candidates.append((k, j, score))
   
   // 按概率升序排序
   sort(drop_candidates, by=score, ascending=True)
   drop_list = drop_candidates[:K]  // 取Top-K
   
   // Step 3: 生成邻居（优先级：AddList在前，DropList在后）
   FOR each (k, j, _) in add_list:
       neighbor = current_pred_assort.copy()
       neighbor[k, j] = 1  // 添加产品j到segment k
       neighbors.append(matrix_to_assignment(neighbor))
   
   FOR each (k, j, _) in drop_list:
       neighbor = current_pred_assort.copy()
       neighbor[k, j] = 0  // 从segment k删除产品j
       neighbors.append(matrix_to_assignment(neighbor))
   
   RETURN neighbors  // 最多2*K个邻居

6. 最终MILP验证
   - 将current_assignment转换为final_pred_assort
   - 对final_pred_assort进行最终MILP求解
   - 得到final_milp_ratio

7. 返回结果
   - final_revenue_ratio = final_milp_ratio
   - search_info = {
         iterations, improvements, lp_solver_calls, 
         milp_solver_calls, K, max_neighbors_per_iter
     }
   - 返回 (final_pred_assort, final_revenue_ratio, search_info)
```

### 关键特点
- **全局Top-K**：K = ceil(sqrt(m))，每轮最多评估2*K个邻居
- **优先级排序**：Add候选按概率降序，Drop候选按概率升序
- **贪婪接受**：找到改进立即接受，不继续评估剩余邻居
- **LP快速评估**：使用LP进行可行性检查和收益评估，速度快
- **MILP验证**：初始和最终使用MILP，中间使用LP加速

## 策略对比总结

| 策略 | 候选集生成方式 | 优化方法 | 计算复杂度 | 特点 |
|------|--------------|---------|-----------|------|
| **FCP** | 阈值方法（threshold=0.0） | 单次MILP | O(预测bundle数) | 简单直接，无迭代 |
| **PCP** | Top-M选择（M=n, threshold=0.5）+ 渐进bundle | 单次优化MILP | O(渐进bundle数) | 链式约束，结构优化 |
| **Global Top-K LS** | 阈值初始 + 全局Top-K邻域 | 迭代Local Search | O(K × iterations) | 迭代优化，LP加速 |

### 核心差异

1. **FCP**: 固定候选集，一次求解
2. **PCP**: 渐进候选集，利用结构约束优化
3. **Global Top-K LS**: 动态邻域搜索，迭代改进

---

## 关键参数说明

### FCP策略
- `threshold = 0.0`: 阈值方法，logits >= 0即选择

### PCP策略
- `M = n`: Top-M选择，每个segment选择前n个产品
- `threshold = 0.5`: 概率阈值，仅保留prob >= 0.5的产品

### Global Top-K LS策略
- `K = ceil(sqrt(m))`: 全局Top-K参数，控制每轮邻居数量
- `max_iterations = 50`: 最大迭代次数
- `tolerance = 1e-6`: 收益改进容忍度
