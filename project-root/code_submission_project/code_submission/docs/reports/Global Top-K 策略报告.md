# Global Top-K 策略详细对比报告

## 一、概述

本报告对比了三种Local Search策略在8个数据集上的实验结果：
1. **原策略 (Segment-based)**: 每轮生成 2*m 个邻域
2. **Global Top-K (K=2*sqrt(m))**: 每轮生成最多 2*K 个邻域，K = ceil(2*sqrt(m))
3. **Global Top-K (K=sqrt(m))**: 每轮生成最多 2*K 个邻域，K = ceil(sqrt(m))

## 二、Revenue Ratio 对比 (收益比率)

| 数据集 | 原策略 | K=2*sqrt(m) | K=sqrt(m) | sqrt vs 原 (差异) |
|--------|--------|-------------|-----------|-------------------|
| m10_n10 | 0.9955 | 0.9949 | 0.9936 | -0.0019 (-0.20%) |
| m20_n10 | 0.9906 | 0.9876 | 0.9884 | -0.0021 (-0.21%) |
| m30_n10 | 0.9826 | 0.9825 | 0.9823 | -0.0004 (-0.04%) |
| BSP_m10n15 | 1.1369 | 1.1369 | 1.1200 | -0.0170 (-1.49%) |
| BSP_m15n15 | 1.1462 | 1.1426 | 1.1221 | -0.0241 (-2.11%) |
| BSP_m20n15 | 1.1662 | 1.1603 | 1.1409 | -0.0253 (-2.17%) |
| **平均** | **1.0775** | **1.0752** | **1.0620** | **-0.0156 (-1.56%)** |

**结论**:
- K=sqrt(m)策略的Revenue Ratio平均下降约1.56%
- 在m_n数据集上差异很小（<0.25%）
- 在BSP数据集上差异稍大（1.5%-3.1%）
- K=2*sqrt(m)策略与原策略基本持平

## 三、Time Ratio 对比 (时间/基准时间)

| 数据集 | 原策略 | K=2*sqrt(m) | K=sqrt(m) | 加速比 (sqrt vs 原) |
|--------|--------|-------------|-----------|---------------------|
| m10_n10 | 0.1170 | 0.0727 | 0.0634 | **1.85x** (-45.8%) |
| m20_n10 | 0.3164 | 0.1049 | 0.0719 | **4.40x** (-77.3%) |
| m30_n10 | 0.6690 | 0.1655 | 0.1118 | **5.98x** (-83.3%) |
| BSP_m10n15 | 8.6859 | 8.5292 | 4.2855 | **2.03x** (-50.7%) |
| BSP_m15n15 | 25.5265 | 20.1424 | 7.7723 | **3.28x** (-69.6%) |
| BSP_m20n15 | 43.4147 | 27.4678 | 11.8595 | **3.66x** (-72.7%) |
| **平均** | **12.33** | **9.73** | **4.26** | **3.21x** (-65.5%) |

**结论**:
- K=sqrt(m)策略平均加速**3.21倍**
- 随着m增大，加速效果更明显：m=30时达到**6倍**加速
- 在BSP数据集上，平均加速**2.5-3.7倍**

## 四、LP调用次数对比

| 数据集 | 原策略 | K=2*sqrt(m) | K=sqrt(m) | 减少率 (sqrt vs 原) |
|--------|--------|-------------|-----------|---------------------|
| m10_n10 | 68.5 | 47.4 | 24.9 | **-63.7%** |
| m20_n10 | 253.7 | 79.0 | 32.8 | **-87.1%** |
| m30_n10 | 544.5 | 107.1 | 44.5 | **-91.8%** |
| BSP_m10n15 | 218.0 | 188.6 | 83.3 | **-61.8%** |
| BSP_m15n15 | 477.3 | 285.7 | 103.9 | **-78.2%** |
| BSP_m20n15 | 756.0 | 365.9 | 144.1 | **-80.9%** |
| **平均** | **368.7** | **200.1** | **83.9** | **-73.5%** |

**结论**:
- K=sqrt(m)策略LP调用次数平均减少**73.5%**
- 这是性能提升的核心原因
- 随着m增大，减少率更高（m=30时达到91.8%）

## 五、Local Search时间对比 (秒)

| 数据集 | 原策略 | K=2*sqrt(m) | K=sqrt(m) | 加速比 (sqrt vs 原) |
|--------|--------|-------------|-----------|---------------------|
| m10_n10 | 0.5523s | 0.3303s | 0.2595s | **2.13x** (-53.0%) |
| m20_n10 | 8.4242s | 2.6844s | 1.6419s | **5.13x** (-80.5%) |
| m30_n10 | 50.4679s | 11.2041s | 6.6882s | **7.55x** (-86.7%) |
| BSP_m10n15 | 1.0530s | 1.0204s | 0.4774s | **2.21x** (-54.7%) |
| BSP_m15n15 | 5.0066s | 3.8552s | 1.4073s | **3.56x** (-71.9%) |
| BSP_m20n15 | 15.8021s | 9.7883s | 3.9169s | **4.03x** (-75.2%) |

- 在大规模问题(m=30)上加速高达**7.55倍**
- Local Search是时间节省的主要来源

## 六、时间组成结构说明

```
Total Time
├── GCN Inference Time (threshold_time)        # GCN模型推理时间
├── Initial MILP Time (initial_milp_time)      # 初始MILP求解时间
└── Local Search Time (local_search_time)      # Local Search总时间
    ├── Initial LP Time                        # 初始LP求解时间
    ├── Iteration Time (total_iteration_time)  # 迭代总时间
    │   ├── Add Candidate Time                 # Add候选构建时间
    │   ├── Drop Candidate Time                # Drop候选构建时间
    │   ├── Neighbor Generation Time           # 邻域生成时间
    │   ├── LP Solve Time                      # LP求解总时间
    │   └── Neighbor Iteration Time            # 邻域遍历时间(不含LP)
    └── Final MILP Time                        # 最终MILP验证时间
```

## 十、随m增长的时间变化

### 表6.2: 增长率分析

| 指标 | m: 10→20 | m: 20→30 | 趋势 |
|------|----------|----------|------|
| Total Time | 6.0x | 4.2x | 超线性但减缓 |
| Initial MILP | 7.1x | 4.7x | 超线性 |
| LP总时间 | 5.9x | 4.0x | 超线性 |
| LP调用次数 | 1.3x | 1.4x | **接近线性** |
| 单次LP时间 | 4.5x | 2.9x | 超线性 |

**关键洞察**:
1. **LP调用次数增长较慢** (1.3x-1.4x)，这是Global Top-K策略的优势
2. **单次LP求解时间增长快** (4.5x-13x)，是时间增长的主因
3. **总时间增长主要由LP求解复杂度驱动**，而非LP调用次数
