| 数据集 | Revenue Ratio差异 | Revenue Ratio变化率 | Time Ratio差异 | Time Ratio变化率 |
|--------|------------------|-------------------|---------------|----------------|

## 关键发现

### 1. Revenue Ratio（收益比率）
- **m10_n10**: K=sqrt(m*n)略好（+0.30%）
- **m20_n10**: K=sqrt(m*n)略好（+0.73%）
- **test_BSP_m10n20**: K=sqrt(m*n)明显更好（+4.00%）

### 2. Time Ratio（时间比率）
- **m10_n10**: K=sqrt(m*n)更慢（+66.32%）
- **m20_n10**: K=sqrt(m*n)更慢（+213.63%）
- **test_BSP_m10n20**: K=sqrt(m*n)显著更慢（+372.40%）

### 3. 权衡分析
- **K=sqrt(m*n)策略**：收益略好，但时间显著增加
- **K=sqrt(m)策略**：时间更快，收益略低
- 对于test_BSP_m10n20数据集，K=sqrt(m*n)的收益提升明显（+4%），但时间成本大幅增加（+372%）

## 结论

**K=sqrt(m)策略**在时间效率上明显优于K=sqrt(m*n)策略，而收益差异较小（除test_BSP_m10n20外）。建议：
- 如果追求**时间效率**：选择K=sqrt(m)
- 如果追求**收益最大化**：选择K=sqrt(m*n)（特别是对于BSP数据集）
