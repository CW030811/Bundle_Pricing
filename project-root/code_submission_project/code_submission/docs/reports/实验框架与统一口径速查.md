# 实验框架与统一口径速查

> 目的：后续任务可直接基于本文件快速对齐实验含义、脚本关系与参数口径。  
> 口径原则：**以代码当前实现为主**，报告/日志作为补充。  
> 扫描时间：2026-02-19

## 1. 项目核心问题与实验主线

- 问题：Mixed Bundling Pricing（混合捆绑定价）优化，目标是最大化收益。
- 输入核心：`m`（segment数）、`n`（产品数）、`unit_cs`、`ship_cs`、`unit_us`、`Ns`。
- 总流程：
  1. GCN 预测候选 bundle（或候选产品链）
  2. MILP 求解定价与分配
  3. （局部搜索类策略）LP 快速评估邻域 + MILP 最终校验

## 2. 策略-脚本映射（主实验入口）

| 策略 | 主脚本 | 核心思想 | 常用输出 |
|---|---|---|---|
| FCP | `test_FCP.py` | 阈值二值化（logit>=0）后直接 MILP | `test_result_threshold_vs_optimal.csv` |
| PCP | `test_PCP.py` | Top-M + progressive bundles 后 MILP | `test_result_topm_vs_optimal.csv`（当前脚本保存行被注释） |
| FCP+LS（原2m邻域） | `test_FCP_LS.py` | 初始MILP + 局部搜索（Add/Drop）+ LP评估 + 终态MILP | `test_result_local_search_mix_*.csv` |
| Global Top-K LS | `LS_Path_Test.py` | 将2m邻域改为全局Top-K，`K=ceil(sqrt(m))` | `test_result_global_topk_sqrtm_*.csv` |
| K选择论证 | `LS_Path_Test_K_Justify.py` | 对比 `original / K=5 / K=10 / sqrt(m) / sqrt(mn)` | `justify_K_*.csv` |
| Cutoff敏感性（FCP） | `sensitivity_cutoff_FCP.py` | cutoff=0.1~0.9 对收益/时间/分类指标影响 | `cutoff_sensitivity_results.csv` |
| Cutoff敏感性（PCP） | `sensitivity_cutoff_PCP.py` | 同上（PCP） | `cutoff_sensitivity_PCP_results.csv` |
| FCP/PCP cutoff对比 | `compare_FCP_PCP_cutoff.py` | 合并并聚合两类敏感性结果 | `cutoff_sensitivity_FCP_PCP_combined.csv` |
| LP→MILP有效性验证 | `verify_lp_milp_improvement.py` | 检验LP改进是否转化为MILP改进 | `LP_MILP_verification_results.json`、`LP_MILP_verification_statistics.json` |
| 求解空间分析 | `analyze_solution_space.py` | 比较 MILP/FCP/PCP 的变量、约束、NumNZs 等 | `solution_space_comparison_*.csv` |

## 4. 关键实验设置（统一口径）

## 4.1 求解精度/时间限制（Gurobi）

- `test_FCP.py`：`MIPGap=1e-2`, `TimeLimit=600s`
- `test_PCP.py`：`MIPGap=1e-2`, `TimeLimit=600s`
- `test_BSP.py`：`MIPGap=1e-2`, `TimeLimit=600s`
- `test_FCP_LS.py`：
  - MILP：`MIPGap=1e-6`, `TimeLimit=600s`
  - LP：`TimeLimit=300s`, `Threads=1`, `Presolve=2`, `Method=-1`
- `LS_Path_Test.py` / `LS_Path_Test_K_Justify.py`：复用 `test_FCP_LS.py` 内 MILP/LP配置

> 说明：项目内并非单一Gap设置。若跨策略做公平比较，需明确是否统一到同一Gap（当前存在 `1e-2` 与 `1e-6` 两套口径）。

## 4.2 局部搜索参数

- 通用：`max_iterations=50`, `tolerance=1e-6`
- 原策略：每轮最多 `2*m` 邻域
- Global Top-K：`K=ceil(sqrt(m))`，每轮最多 `2*K` 邻域
- Drop候选过滤：仅考虑 `prob>=0.5` 的已选产品

## 4.3 Cutoff敏感性参数

- Cutoff集合：`[0.1, 0.2, ..., 0.9]`
- FCP敏感性：每个MB数据集抽样30（总90）
- PCP敏感性：每个MB数据集抽样10（总30）
- 随机种子：`np.random.seed(42)`, `torch.manual_seed(42)`

## 4.4 K-Justify参数

- 比较策略：`original`, `K_const_5`, `K_const_10`, `K_sqrt_m`, `K_sqrt_mn`
- 每数据集：`max_samples=100`
- 随机种子：`np.random.seed(42)`, `torch.manual_seed(42)`

## 4.5 LP-MILP验证参数

- 数据集：6组（`m10/m20/m30_n10` + `BSP m10n15/20/25`）
- 每组10样本，总60样本
- 特殊样本选择：`m10_n10_sample_100` 跳过 `sample_id=5`，改用 `sample_id=10`（脚本中明确说明，原因是疑似Gap导致异常）

## 7. 关键实验结论（供后续任务快速定位）

- Global Top-K（`K=ceil(sqrt(m))`）：
  - 与原2m策略相比，收益接近，LP调用与时间显著下降（见 `实验对比总结报告.md`, `Global_TopK_Summary.csv`）。
- Cutoff敏感性：
  - FCP：cutoff升高通常提升收益、降低误选，但漏选风险上升。
  - PCP：收益总体高且稳定，但时间比FCP更高（见 `Cutoff_Sensitivity_实验报告.md`）。
- LP→MILP验证：
  - 总体转化率高（统计文件显示约97.5%），LP作为邻域筛选近似可靠，但存在少量未转化案例。
- 求解空间：
  - FCP显著压缩变量/约束；PCP在大m下约束增长明显（progressive相关约束导致）。
