# 060426 Report: BSP+FCP / CPBSD-A+FCP Hybrid OOS 实验

## 策略思路

### 动机

FCP 的 OOS 劣势来自 menu coverage 不足（34-51 bundles vs 2^N）。BSP 和 CPBSD-A 都能提供全覆盖定价：
- **BSP**：N+1 个 size prices，`P(S) = q_{|S|}`
- **CPBSD-A**：2N 个参数 `(p_n, d_s)`，`P(S) = Σp_n - |S|·d_{|S|}`

### Hybrid 策略

以 BSP 或 CPBSD-A 的全覆盖定价为底层，用 FCP 的 per-bundle prices 覆盖其中一部分 bundles。

**Greedy Safe Max**：对 FCP price > base price 的 anchor bundles，逐个替换，仅在不破坏 subadditivity 时保留。

Subadditivity 安全判定（解析式，O(N) per bundle）：
- BSP base：`P_new(A) <= min_{s1+s2=|A|} [q_{s1} + q_{s2}]`
- CPBSD-A base：`P_new(A) <= Σp_n(A) - max_{s1+s2=|A|} [s1·d_{s1} + s2·d_{s2}]`

### 预期

提价方向（max）在 OOS 上**不可能改善 revenue**——提高 bundle 价格只会降低 customer surplus，customer 要么转向更便宜的 base-priced bundle，要么不变。这是结构性结论。

## 结论

### Hybrid 表现汇总

| Setup | BSP+FCP vs BSP | CPBSD-A+FCP vs CPBSD-A |
| --- | ---: | ---: |
| N10 zero | −0.14 | +0.01 |
| N10 hvhm | +0.07 | −0.03 |
| N30 zero | −0.80 | −0.04 |
| N30 hvhm | +0.03 | −0.03 |

- **BSP+FCP hybrid**：2/4 setup 略优于 BSP，2/4 略劣。改善幅度极小（<0.1），N30 zero 上反而恶化 0.80。
- **CPBSD-A+FCP hybrid**：4/4 setup 均 ≤ CPBSD-A，最大差距 −0.04。

### 结构性结论

**在全覆盖定价（BSP 或 CPBSD-A）基础上，用 FCP 的高价 bundles 做 max 替换，不能系统性改善 OOS revenue。**

原因：
1. 提价降低 customer surplus → customer 转向 base-priced 替代 bundle
2. 全覆盖 base 已经为每个 customer 提供了最优选择，FCP 的 bundle-specific 高价只会把 customer 赶走
3. FCP 的 per-bundle pricing 优势体现在 in-sample（表达力强），但这种优势无法通过 post-hoc overlay 迁移到 OOS

## Random Cost Setup 实验

### Cost 设计

在 `generate_data_CPBSD.py` 中新增两种 cost scenario：

- **`random_ind`**：`c_n ~ U[0, 10]`，与 valuation 完全独立，scale 和 valuation mean [1,10] 一致
- **`random_corr`**：`c_n = mean_val_n × U[0.3, 0.9] + N(0, 0.5)`，与 valuation mean 正相关但有随机扰动

### Joint FCP+BSP MILP 说明

Joint MILP 同时优化 FCP bundle prices 和 BSP size prices，加入跨菜单 anti-arbitrage 约束（C1-C14，详见 `joint_fcp_bsp_formulation.md`）。

- **Joint**: 含 C12 约束（`p_i ≤ q_{|S_i|}`，FCP ≤ BSP 同 size）
- **Joint-noC12**: 移除 C12，仅保留 C13 split anti-arbitrage

N=5 使用 full 2^5=32 assortments；N=10/30 使用 GCN-pruned assortments（与 FCP 相同）。

> **注意**：以下所有数据来自 clean rerun（`run_joint_clean_rerun.py`），所有方法在同一 instance 文件上运行，使用同一 OOS 样本，确保 apple-to-apple 比较。

### 关键发现

1. **`random_ind` 上 FCP 最强**：N=5 OOS=9.37 和 N=10 OOS=16.37 均领先。独立随机 cost 下 bundle-specific pricing 的优势明显，Joint 反而因为 IC 约束过拟合而略逊于纯 FCP。

2. **`random_corr` 上 CPBSD-A 和 BSP 领先**：N=5 CPBSD-A=9.90 最优，N=10 BSP=Joint-noC12=18.97 并列最优。相关 cost 下参数化定价（BSP 的 size pricing、CPBSD-A 的 component pricing）的正则化效果优于 bundle-specific 方法。

3. **Joint 在所有 setup 上都不是 OOS 最优**：联合优化的 IC (C7) 约束（K²=2500 条）导致过拟合。Joint InS 通常最高，但 OOS 泛化差。

4. **noC12 一致优于或等于 C12**：移除 `p_i ≤ q_{|S_i|}` 后 OOS 在所有 setup 上不变或提升，说明该约束不必要。

5. **N=10 `random_corr` 各方法极其接近**：OOS 范围仅 18.84-18.97（spread=0.13），说明在这个 setup 下定价策略差异很小。

6. **CPBSD-A 在 random cost 上表现差**：特别是 N=10/30 random_ind，component pricing 的 preset ranking 在随机 cost 下严重失效。
